  | 규칙 이름 | 수준 | 역할 | 실제 동작 예시 |
  | --------------------------------------------- | --------- | --------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- |
  | **`@typescript-eslint/no-unused-vars`** | `"warn"` | 사용하지 않는 변수나 매개변수에 경고 표시 | `ts const x = 10; // 사용 안하면 ⚠️ 경고` |

프로젝트 구조를 도메인별로 분리: 리팩토링 했다.

- 변경된 구조
  src/
  ├── app.ts # Express 앱 설정 (미들웨어, 라우터 등록)
  ├── server.ts # 서버 시작 (포트 listen)
  **[역할 분리]**
  app.ts
  - Express 앱 생성 및 설정
  - 미들웨어 등록
  - 라우터 등록
  - 에러 핸들러 등록
  - export default app - 앱 객체만 export
    server.ts
  - 환경 변수 로드
  - DB 초기화
  - app을 import해서 서버 시작 (app.listen)
  - 실제 서버 구동 진입점
    **[왜 이렇게 분리?]**
  1. 테스트 용이성 - app.ts를 테스트할 때 서버를 실제로 띄우지 않고 테스트 가능
  2. 관심사 분리 - 앱 설정과 서버 실행 로직을 명확히 분리
  3. 재사용성 - app을 다른 곳에서 import해서 사용 가능
